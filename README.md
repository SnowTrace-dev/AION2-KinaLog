# 키나로그 V1.3.2

## 실행 주소

https://snowtrace-dev.github.io/AION2-KinaLog/

## 개요

키나로그는 AION2 주간 플레이를 한 화면에서 관리하기 위한 개인용 체크리스트 겸 수익 기록 도구입니다.

캐릭터별 오드 에너지, 충전 에너지, 숙제, 원정, 초월, 성역 보상, 주간 수익, 추가 수익, 지출, 역대 수익, 성장 기록, 스티그마 레벨, 메모를 브라우저 LocalStorage에 저장해 관리합니다.

## 포함 파일

- `index.html`
- `assets/Logo.png`
- `assets/favicon.ico`
- `assets/favicon.png`
- `assets/Small.png`
- `assets/Large.png`

## 별도 추가 파일

아래 폰트 파일은 직접 `assets` 폴더에 추가하면 적용됩니다.

- `assets/NEXONLv1GothicRegular.ttf`

폰트 파일이 없어도 브라우저 기본 한글 폰트로 정상 실행됩니다.

## 주요 기능

### 1. 캐릭터 관리

- 최대 6개 캐릭터 등록
- 캐릭터 삭제
- 캐릭터 순서 변경
- 전투력 입력 및 수정
- 오드 에너지 / 충전 에너지 직접 수정
- 캐릭터별 주간 수익 표시

### 2. 오드 에너지 관리

- 기본 오드 에너지와 충전 에너지를 분리 표시
- `오드 에너지 000 / 840 (+000)` 형식으로 표시
- +10 / +40 충전 에너지 버튼 제공
- 원정 / 초월 / 성역 보상 수령 시 콘텐츠별 오드 자동 차감
- 오드가 부족하면 보상 수령 방지
- 3시간 단위 자연 회복 반영
- 보상 취소 시 사용 오드 환급
- 일반 오드 환급은 최대 오드 상한을 넘지 않도록 제한

### 3. 숙제 체크리스트

- 캐릭터별 숙제 등록
- 일반 숙제는 항목 클릭 또는 체크박스로 완료/해제
- 원정 / 초월은 체크박스와 보상 버튼 중심으로 관리
- 성역은 보상 수령 시 자동으로 완료 체크
- 보상 기록이 있는 숙제는 먼저 보상 취소 후 삭제 가능

### 4. 원정 / 초월 관리

- 캐릭터별 원정 / 초월 항목 등록
- 캐릭터별 원정 35회 / 초월 28회 제한 제거
- 원정 / 초월 티어 선택
- 보상 버튼으로 수익 자동 계산
- 취소 버튼으로 마지막 보상 기록 롤백
- 주간 누적 플레이 보상 횟수 자동 집계
- 챕터1 보상표 반영
- 120 오드 콘텐츠의 오드 효율 계산 반영

### 5. 성역 관리

- `[성역] 루드라`
- `[성역] 바고트`
- `[성역] 칼드릭스`
- 성역 보상은 주간 1회만 기록
- 성역 보상 수령 시 오드 80 차감 및 수익 자동 계산
- 성역 보상 수령 시 해당 숙제 자동 완료 체크
- 성역 보상 취소 시 완료 체크 해제 및 보상 기록 롤백
- 성역 취소 환급 시 일반 오드가 최대치를 넘지 않도록 제한

### 6. 주간 누적 플레이 보상 조정

- 원정 주간 누적 횟수 표시
- 초월 주간 누적 횟수 표시
- 주간 보상 계산용 카운터 자동 갱신

### 7. 주간 수익

- 원정 수익 집계
- 초월 수익 집계
- 성역 수익 집계
- 일반 키나 / 각인 키나 / 총합 분리 표시
- 항목별 횟수 표시
- 추가 수익과 지출 반영
- 전체 주간 수익 요약 표시

### 8. 추가 수익 / 지출

- 추가 수익 직접 입력
- 지출 직접 입력
- 같은 항목명 자동 합산
- 프리셋 저장
- 프리셋 삭제
- 각인 키나 항목명은 각인 수익으로 분리 집계

### 9. 역대 수익

- 주차별 수익 자동 저장
- 시즌3 / 챕터1 주차명 표시
- 일반 키나 / 각인 키나 / 총합 표 형태로 표시
- 주차별 상세 내역 펼쳐보기
- 원정 / 초월 / 성역 / 추가 수익 / 지출 상세 분리

### 10. 오드 효율표

- 원정, 초월, 성역 콘텐츠별 오드 비용 표시
- 일반 키나 / 각인 키나 / 총합 표시
- 10 오드당 기대 수익 계산
- 효율 높은 순서로 정렬

### 11. 성장 / 스티그마

- 캐릭터별 전투력 성장 기록 표시
- 주간 평균 전투력 상승량 표시
- 성장 탭의 캐릭터 카드를 가로 드래그로 이동
- 스티그마명과 스티그마 레벨 입력
- 캐릭터별 스티그마 최대 13개 관리
- 스티그마 레벨 기준 남은 샤드 계산
- 회랑 기록과 각성전 점수 기반 주간 스티그마 예상 수급량 표시
- 완료 예상 주차와 예상 일자 표시
- 각성전 점수 구간 선택
- 회랑 결과 입력 기록 기반 평균 계산

### 12. 메모

- 최대 3개 메모 카드 생성
- 메모 / 체크 / 구분선 블록 추가
- 메모 카드 및 메모 블록 정렬 모드 지원
- 체크 항목 완료 표시
- 긴 메모 입력 시 자동 높이 조정

### 13. 데이터 백업 / 복원

- JSON 백업
- JSON 복원
- 백업 파일명: `kinalog_backup_YYYY-MM-DD.json`

## 저장 방식

키나로그는 별도 서버를 사용하지 않습니다.

모든 데이터는 브라우저의 LocalStorage에 저장됩니다.

따라서 다른 PC, 다른 브라우저, 다른 기기에서는 데이터가 자동 공유되지 않습니다.

중요한 데이터는 주기적으로 백업하는 것을 권장합니다.

## 배포 방법

1. 저장소에 파일 업로드
2. `index.html`을 루트에 배치
3. `assets` 폴더를 함께 업로드
4. 필요한 경우 `assets/NEXONLv1GothicRegular.ttf`를 직접 추가
5. GitHub Pages 주소 접속

## V1.3.2 변경 사항

### 버그 수정

- 구버전 숫자 보상 또는 `aetherCost`가 없는 저장 보상이 오드 80을 새로 만드는 문제 수정
- `aetherCost: 0` 보상이 `amount || 80` 처리로 80 오드 소모처럼 바뀌는 문제 수정
- `spendAether()`에서 0 오드 비용은 실제 소모 없이 기록되도록 수정
- 보상 취소 환급 시 일반 오드가 최대 오드 상한을 초과해 증가하는 문제 수정
- 성역 완료 후 자연 회복이 된 상태에서 취소할 때 오드가 80씩 생기는 현상 방지
